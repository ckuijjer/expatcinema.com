# Expat Cinema - Agent Guide

This is the canonical repository guide for coding agents working in this repo.

## Session Setup

Always run `nvm use` at the start of a session to activate Node 24 from `.nvmrc`.

```bash
nvm use
```

## Project Overview

Expat Cinema aggregates foreign-language movie screenings with English subtitles from Dutch cinemas. The system:

1. scrapes cinema websites in `cloud/`
2. resolves titles to stable movie ids and metadata
3. writes public JSON to S3
4. builds a static Next.js site in `web/`

Live site: https://expatcinema.com

## Repository Structure

```text
expatcinema.com/
├── cloud/      AWS CDK + Lambda scrapers + metadata resolution
├── web/        Next.js static site deployed to GitHub Pages
├── scripts/    Local analysis/download utilities
├── backend/    Minimal package, not central to the current app
└── data/       Local data storage (gitignored)
```

## Current Architecture

### Cloud

- Runtime: Node.js 24
- Infra: AWS CDK in TypeScript
- Storage:
  - public S3 bucket for `screenings.json`, `movies.json`, `title-matches.json`
  - private S3 bucket for per-scraper outputs and review artifacts
  - DynamoDB for analytics only
- Scraping:
  - `got` and `x-ray` for HTML/API scrapers
  - Puppeteer + Chromium for JS-heavy sites
- Metadata:
  - TMDB-based movie resolution
  - manual overrides in `cloud/metadata/manualTitleOverrides.ts`
  - review artifacts for ambiguous and unmatched title resolution cases

### Web

- Next.js 16 static export
- React 19
- GitHub Pages deployment
- Reads public JSON generated by the cloud pipeline

## Core Data Model

### Screening

Scrapers return:

```ts
type Screening = {
  title: string
  url: string
  cinema: string
  date: Date
}
```

Public `screenings.json` keeps that shape and adds only:

```ts
movieId?: string
```

### Movies

Public `movies.json` contains one entry per canonical movie, keyed by `movieId` values such as `tmdb:194`.

## Important Outputs

Public:

- `screenings.json`
- `movies.json`
- `title-matches.json`

Private:

- per-scraper raw outputs
- `review/ambiguous-movies/<timestamp>.json`
- `review/unmatched-movies/<timestamp>.json`

## Local Workflows

### Install dependencies

```bash
pnpm install
```

### Install local Chromium

```bash
cd cloud
pnpm run install-chromium
```

### Run scrapers locally

Use `cloud/.env.local`.

```bash
cd cloud
pnpm run scrapers:local
```

Local output is written to:

- `cloud/output/<PUBLIC_BUCKET>/screenings.json`
- `cloud/output/<PUBLIC_BUCKET>/movies.json`
- `cloud/output/<PUBLIC_BUCKET>/title-matches.json`
- `cloud/output/<PRIVATE_BUCKET>/...`

### Run prod scrapers

```bash
cd cloud
pnpm run scrapers:prod
```

Requires a valid AWS session for the `casper` profile.

### Tail production scraper logs

```bash
cd cloud
AWS_PROFILE=casper aws logs tail /aws/lambda/expatcinema-prod-scrapers --since 30m --follow --region eu-west-1
```

## Deployment Workflows

### Cloud

Workflow:

- `.github/workflows/cloud.yml`

Triggers:

- push to `main`

- `.github/workflows/web.yml`

## Scraper Conventions

- Use `luxon` with Europe/Amsterdam timezone handling.
- Return absolute screening URLs.
- Keep scraper output titles as scraped unless there is scraper-specific programme noise that should clearly be removed.
- Prefer scraper-level cleanup for recurring cinema-specific prefixes and suffixes.
- Keep scraper files lowercase and cinema-named, e.g. `kinorotterdam.ts`.

## Metadata Conventions

- Stable movie identity is `movieId`, not title string.
- `screenings.json` keeps original screening titles.
- Resolver/match details belong in `title-matches.json` and review artifacts, not in public screenings rows.
- Manual corrections should go in `cloud/metadata/manualTitleOverrides.ts`.
